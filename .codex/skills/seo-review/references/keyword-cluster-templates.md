# Keyword Cluster Templates

Use these patterns to build a keyword cluster that matches the page intent.
Pick 5-10 total queries (1 primary, 3-8 secondary).

## Generic Cluster Patterns

| Type | Pattern | Example ("data validation") |
| --- | --- | --- |
| Primary | [topic] | data validation |
| What is | what is [topic] | what is data validation |
| How to | how to [verb] [topic] | how to implement data validation |
| How does | how does [topic] work | how does data validation work |
| Benefits | why use [topic] | why use data validation |
| Examples | [topic] examples | data validation examples |
| Best practices | [topic] best practices | data validation best practices |
| vs | [topic] vs [related] | data validation vs sanitization |
| Troubleshooting | [topic] errors | data validation errors |
| Tools | [topic] tools | data validation tools |

## Page-Type Variations

- Concept/definition pages: emphasize "what is" and "how it works".
- How-to guides: emphasize "how to" and steps.
- Product/feature pages: emphasize benefits, use cases, and alternatives.
- Comparison pages: emphasize "vs" and decision criteria.
- Troubleshooting pages: emphasize errors, fixes, and edge cases.

## Selection Rules

- Keep the primary keyword short, clear, and high intent.
- Use singular/plural variants if natural in the domain.
- Prefer user language over internal jargon.
- Avoid stuffing exact-match phrases; use variations in copy.
